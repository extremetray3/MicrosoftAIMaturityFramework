Your community needs a digital home and tools for collaboration. Microsoft provides a rich set of platforms, often at little extra cost because many organizations already use Microsoft 365.

## <iconify-icon icon="fluent-emoji-flat:hammer-and-wrench" width="24" height="24" style="vertical-align: middle; margin-right: 0.2em;"></iconify-icon> Infrastructure and Tooling

### Viva Engage
[Viva Engage](https://learn.microsoft.com/en-us/viva/engage/overview) is ideal for broad, open discussions and updates:
- Create a **dedicated community group** for Copilot Studio
- Seed with an introduction post stating the community's purpose
- Use **polls**, **announcements**, and **pinned posts** for FAQs
- Leverage Viva Engage's **event capabilities** for meetups and RSVPs

{: .note }
> If your organization is more Teams-centric, replicate this approach by creating a public team or channel for Copilot Studio enthusiasts.

### SharePoint Communication Site
Set up a SharePoint site or intranet page as the community's **knowledge hub**:
- **Announcements** — Latest community news, upcoming events
- **Training resources** — Links to Microsoft Learn paths, internal recordings
- **Success stories** — Case studies of internal agent solutions
- **How-to guides and FAQs**
- **Governance policies** — Environment and data policy guidelines
- **Community guidelines** — How to get involved, code of conduct

### Microsoft Teams
- **Live events:** Host trainings, office hours, and virtual meetups
- **Core team coordination:** Private channel for leaders to manage plans and files
- **Project collaboration:** Smaller interest groups or project teams

### Microsoft Forms
- **Event feedback:** Reusable post-event feedback forms with branching
- **Community surveys:** Quick polls and preference gathering
- **Assessment tools:** Lightweight skill-level assessments or knowledge checks

### Other Channels
- **Email newsletters** for users who prefer email
- **Digital signage / intranet banners** to promote events
- **Viva Glint or Viva Insights** for collaboration measurement if available

## <iconify-icon icon="fluent-emoji-flat:link" width="24" height="24" style="vertical-align: middle; margin-right: 0.2em;"></iconify-icon> Partnerships and Amplification

### <iconify-icon icon="fluent-emoji-flat:link" width="24" height="24" style="vertical-align: middle; margin-right: 0.2em;"></iconify-icon> Internal Partnerships

- **IT and security teams:** Include early. Ensure community activities align with IT policies. Turn potential blockers into supporters.
- **Data and analytics teams:** Collaborate on crossover use cases (e.g., AI agents that surface BI insights). Cohost joint events.
- **Business units and process teams:** Help identify high-value use cases. Get domain sponsorship for targeted hackathons.
- **Microsoft 365 champions / modern work community:** Join forces for joint sessions showing how technologies complement each other.
- **Executive partnerships:** Identify senior leaders passionate about innovation. Invite them as executive champions and active participants.

### External Networks

- **Microsoft user groups:** Find local or virtual Power Platform user groups. Invite external speakers for fresh perspectives.
- **Cross-company exchanges:** Host friendly collaboration or knowledge exchange with other organizations' communities.

### Amplification Tactics

- **Publicize success** through internal channels: success story emails, intranet articles, town hall shout-outs
- **Share ideas across communities:** Propose a "Community Fair" where each community presents lightning talks
- If appropriate, share externally on **LinkedIn** or **Microsoft Tech Community** forums

### Branding and Identity

- **Name and identity:** Use a catchy name that resonates (e.g., "Future of Work Guild" or "AI Agents Alliance")
- **Visual design:** Create a logo consistent with Copilot Studio and company branding
- **Elevator pitch:** Prepare a brief, consistent description for partners and potential members
